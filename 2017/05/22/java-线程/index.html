java 线程

发表于 2017-05-22 | 分类于 java | | 浏览

线程创建

在 Java 程序语言 中， 可通过 系统 提供 的 编程 接口 去 创建 线程， 创建 线程 的 方法 有 两种： 一种 是 通过 实现 Runnable 接口 的 方式 来 创建 线程， 另一种 是 通过 继承 Thread 类 来 创建 线程。

  1. 通过 Runnable 接口 的 方式 创建 线程
  2. 通过 继承 Thread 类 来 创建 线程

线程的状态

线程 包括 5 种 状态： 新建 状态、 就绪 状态、 运行 状态、 阻塞 状态 和 死亡 状态。 下面 分别 详细 介绍 这 5 种 状态。

线程的优先级

setPriority
1:表示优先级最低
10：表示优先级最高
5：表示默认优先级

线程的休眠与唤醒

线程 的 休眠， 是指 线程 暂时 处于 等待 的 一种 状态， 通俗 地说， 就是 线程 暂时 停止 运行 了。 要 达到 这种 功能 需要 调用 Thread 类 的 sleep() 方法。 sleep() 方法 可以 使 线程 在 指定 的 时间， 处于 暂时 停止 的 状态， 等到 指定 时间 结束 后， 暂时 停止 状态 就会 结束， 然后 继续 执行 没有 完成 的 任务。

线程 的 唤醒 是指， 使 线程 从 休眠 等待 状态 进入 可执行 状态， 可以 通过 调用 方法 interrupt() 来 实现。

线程让步

所谓 线程 让步， 就是 使当 前 正在 运行 的 线程 对象 退出 运行 状态， 让 其他 线程 运行， 其 方法 是 通过 调用 yield() 方法 来 实现。

线程同步

过， 线程 的 运行 权 通过 一种 叫 抢占 的 方式 获得。 一个 程序 运行 到 一 半时， 突然 被 另一个 线程 抢占 了 运行 权， 此时 这个 线程 数据处理 了 一半， 而 另一个 线程 也在 处理 这个 数据， 那么 就会 出现 重复 操作 数据 的 现象， 最终 导致 整个 系统 的 混乱。synchronized
